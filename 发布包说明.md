# 茶叶监测系统发布包说明

## 1. 发布包内容

发布包面向最终使用人员，解压后即可运行，一般不需要重新编译项目。

典型发布包通常包含以下内容：

- `TeaMonitorFusion.exe`
- Qt 运行库与插件目录
- `SeaSDK.dll`
- `启动茶叶监测系统.vbs`
- 本说明文档

## 2. 推荐启动方式

推荐优先双击以下文件启动软件：

- `启动茶叶监测系统.vbs`

这样可以尽量避免命令行黑窗口干扰。

如果需要，也可以直接双击：

- `TeaMonitorFusion.exe`

## 3. 适用场景

当前发布包适用于以下场景：

- 软件界面演示
- 使用模拟数据进行流程验证
- 已具备对应运行环境的设备联调

## 4. 真实设备运行前的额外条件

如果需要连接真实设备，仅复制发布包通常还不够，目标电脑往往还需要提前安装对应厂商驱动、运行库或 SDK。

### 4.1 如海广电光谱仪

目标电脑通常需要具备：

- `SeaSDK.dll`
- 对应设备驱动

当前发布包可携带 `SeaSDK.dll`，但如果系统缺少驱动，仍可能无法识别真实设备。

### 4.2 海洋光学光谱仪

如果需要在目标电脑上直连真实设备，通常还需要：

- 安装 `OceanDirect` 运行环境或 SDK
- 保证驱动与依赖库完整

### 4.3 工业相机

工业相机当前按 `eBUS SDK` 方向预留。

如果需要连接真实工业相机，目标电脑通常还需要：

- `Pleora eBUS Runtime / SDK`
- 正确的网卡或相机驱动环境

## 5. 使用步骤

1. 将整个发布目录完整复制到目标电脑
2. 不要只复制 `TeaMonitorFusion.exe`，需要保留同级目录中的 `.dll` 和 Qt 插件文件夹
3. 推荐优先运行 `启动茶叶监测系统.vbs`
4. 如果真实设备连接失败，请先检查目标电脑是否已经安装对应驱动和厂商运行环境

## 6. 目录要求

请保持发布包目录结构完整，不要删除或移动以下内容：

- `platforms`
- `styles`
- `imageformats`
- `iconengines`
- `generic`
- `networkinformation`
- `tls`
- `translations`
- 各类 Qt `.dll`
- `SeaSDK.dll`

否则程序可能无法启动，或启动后缺少图像、主题、网络或设备相关功能。

## 7. 常见问题

### 7.1 双击后无法启动

优先检查：

- 是否误删了 `platforms` 文件夹
- 是否只复制了 `TeaMonitorFusion.exe`
- 是否被安全软件拦截

### 7.2 能打开界面，但无法连接设备

这通常不是发布包损坏，而是目标电脑缺少：

- 厂商驱动
- SDK 运行环境
- 正确的设备连接条件

### 7.3 界面图标或样式显示异常

请确认发布包中的 Qt 插件目录和 `.dll` 文件仍然完整。
