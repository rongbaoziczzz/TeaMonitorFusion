# 茶叶监测系统重构版

该目录是基于 `Qt_test001` 与 `Tea-monitor-system-main` 思路重构的新独立项目，不会改动原始工程。

## 当前重构目标

- 统一启动入口：先选设备，再进工作台。
- 明确区分两套光谱仪与两套 SDK。
- 工业相机优先保留 eBUS 接入方向。
- 全界面尽量中文化。
- 增加“返回设备选择”回退流程，提升可操作性。

## 已接入的设备架构

### 光谱仪

- 海洋光学光谱仪
  - 波段：`1000-1700 nm`
  - SDK：`OceanDirect SDK`
  - 现状：代码层已完成独立适配类与参数接口封装。
  - 说明：当前项目使用的是 `Qt 6.10.1 + MinGW`。本机虽然已安装 OceanDirect SDK，但官方库在当前 MinGW 构建下无法直接链接，因此运行时会给出中文提示。后续若切换到 `MSVC Qt 套件`，更适合继续完成真实直连。

- 如海广电光谱仪
  - 波段：`400-1100 nm`
  - SDK：`SeaSDK`
  - 现状：已按原 `Qt_test001` 的 SeaSDK 方式接入，并保留积分时间、平滑、平均次数、实时采集等能力。

- 光谱模拟器
  - 用于无设备环境下的软件联调、培训和界面演示。

### 工业相机

- eBUS 工业相机
  - SDK 方向：`Pleora eBUS SDK`
  - 现状：已加入独立适配类和 CMake 自动检测逻辑。
  - 本机检查结果：当前仅明确检测到 `eBUS Player`，未检测到开发头文件 `PvSystem.h`，因此当前版本会在连接时给出中文提示，而不会伪装成已接入真实相机。

- 工业相机模拟器
  - 用于流程演示、界面联调和非现场开发。

## 交互优化

- 启动页展示设备名称、波段范围和对应 SDK。
- 工作台顶部新增 `返回设备选择`，可随时回到启动页重新选设备。
- 工作台主操作收敛为：
  - `连接设备`
  - `开始监测`
  - `停止监测`
  - `导出光谱 CSV`
  - `保存当前图像`
- 光谱页保留暗参考与白参考采集。
- 所有高频提示、状态和日志尽量改为中文。

## 构建方式

```powershell
cmake -S . -B build -G "MinGW Makefiles"
cmake --build build
```

构建完成后，可执行文件位于：

- `build/TeaMonitorFusion.exe`

## 当前建议

如果你下一步希望真正打通“海洋光学光谱仪 + eBUS 工业相机”的正式生产版，建议优先确认两件事：

1. Qt 是否切换到 `MSVC` 套件。
2. 本机是否安装完整的 `Pleora eBUS SDK` 开发包，而不只是 `eBUS Player`。
